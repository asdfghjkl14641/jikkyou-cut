# コメント分析画面: MVP 設計仕様書

## 目的

URL動画DL後、編集対象となる「視聴率が高くなりそうな区間」をユーザが選択するための画面を提供する。
これは現在のワークフロー(URL DL → 編集画面)に**1ステップ挟む**形になる。

## 新ワークフロー

```
1. URL入力 → 動画DL(既存実装済)
2. ★ コメント分析画面 ★ (新規)
   - チャットリプレイ取得
   - 盛り上がりスコア計算 + グラフ表示
   - ユーザが「ここを切り抜く」区間を選択
3. 選択した区間が編集画面に渡される(既存実装の編集画面)
4. 文字起こし → 編集 → 書き出し(既存)
```

## 対応プラットフォーム(MVP)

- **YouTube アーカイブ配信のチャットリプレイ**(主戦場)
- Twitch VOD は将来対応(MVP では一旦対象外、設計時に拡張余地は残す)

## 盛り上がりスコア(MVP の合成式)

3要素を統合した「盛り上がりスコア」を時系列グラフで表示。

### 要素1: コメント密度
- 単位時間あたりのコメント数(分速 or 秒速)
- 一定窓(例: 30秒)で移動平均

### 要素2: 特定キーワード出現
- リスト例: 草、w、www、888、すごい、やばい、神、www、すげー、まじか
- 重み付け可(将来カスタマイズ可)
- 出現数を時系列で集計

### 要素3: 視聴者数増加
- YouTube Data API で同時視聴者数の時系列を取得(取得可否を実装時に再確認)
- 増加速度(微分)を時系列で集計
- 取得困難な場合は MVP では省略(コメント密度+キーワードの2要素で開始)

### 統合スコア
```
score(t) = w1 * 正規化(コメント密度(t))
        + w2 * 正規化(キーワード出現(t))
        + w3 * 正規化(視聴者増加(t))
```
重み w1, w2, w3 はデフォルト 1:1:1、将来ユーザ調整可。

### 主要要素

1. **動画タイトル表示**(動画情報をyt-dlpから取得)
2. **盛り上がりスコアグラフ**(時系列、Chart.js 等のライブラリを使用)
3. **個別グラフトグル**(上級者向け、3要素を別グラフ表示)
4. **自動検出された切り抜き候補リスト**(スコア高い区間を自動抽出)
5. **手動区間選択**(グラフ上をドラッグで選択)
6. **「編集画面へ」ボタン**(選択区間を編集画面に渡す)

## 技術設計

### チャットリプレイ取得

#### YouTube
```bash
yt-dlp --write-subs --sub-langs live_chat --skip-download <URL>
```

出力される `.live_chat.json` をパース:
- 各メッセージの timestamp_text・message・author を抽出
- 時系列順でソート
- 解析処理に渡す

実装場所: `src/main/commentAnalysis/youtubeChatExtractor.ts`(新規)

### スコア計算

実装場所: `src/main/commentAnalysis/scorer.ts`(新規)

```typescript
type CommentEvent = {
  timestamp: number;  // 秒
  message: string;
  author: string;
};

type ScoreSeries = {
  timestamps: number[];     // 各時点(秒)
  commentDensity: number[];  // 各時点のコメント密度
  keywordHits: number[];     // 各時点のキーワード出現数
  viewerGrowth: number[];    // 各時点の視聴者増加(取得可なら)
  combinedScore: number[];   // 統合スコア
};

function analyzeComments(events: CommentEvent[]): ScoreSeries { ... }
```

### グラフ表示

実装場所: `src/renderer/src/components/CommentAnalysisView.tsx`(新規)

ライブラリ: Chart.js (既存依存) or recharts(既存依存)
- Chart.js でラインチャート、x軸=時間、y軸=スコア
- 区間ハイライト(候補区間 or ユーザ選択中区間を背景塗りつぶし)
- ドラッグで区間選択(イベントハンドラで開始・終了位置取得)

### 区間選択 → 編集画面遷移

選択された startSec/endSec を持って編集画面に遷移する設計を新規導入。
既存の編集画面は「動画ファイル全体」を対象にしているため、区間指定で開くフローを追加。

実装案:
- ProjectFile に `clipRange?: { startSec, endSec }` を追加(jcut.json 拡張)
- 動画読み込み時、clipRange があればその区間を編集対象に(Gladia 文字起こしも当該区間のみ)
- これにより、コメント分析画面で選択 → 編集画面で「その区間だけが操作対象」になる

詳細は実装時に再検討。

## 実装段階(MVP → 拡張)

### MVP(最初に実装)
1. yt-dlp でチャットリプレイ取得
2. コメント密度 + キーワード出現で2要素スコア計算
3. グラフ表示 + 自動候補リスト
4. 手動ドラッグ区間選択
5. 編集画面に区間情報を渡す

### Phase 2(MVP安定後に追加)
- YouTube Data API で視聴者数増加データ取得
- スコアアルゴリズムの調整(重み変更UI)
- Twitch VOD のチャットリプレイ対応
- 個別グラフ表示の充実

### Phase 3(将来)
- IDEAS.md の他項目(感情波形、伏線検出 等)と統合
- AIによる候補区間の質的判定
- ユーザフィードバックでスコアアルゴリズム学習

## 法的・利用上の注意

- チャットリプレイ取得は yt-dlp の機能であり、公開動画のメタデータ取得に該当
- 既存の利用規約(URL DL機能で同意済み)に「権利者から許諾を得たコンテンツ」の記載あり、コメント分析もその範囲内で運用
- YouTube Data API を使う場合、API キー管理 + レート制限を考慮

## 開放されている設計判断

実装着手時に確定する項目:
- グラフ用ライブラリ(Chart.js vs recharts)
- 自動候補抽出のしきい値(スコア > X 秒)
- キーワードリストの初期値(草・w・wwwの追加範囲)
- 区間選択の UX(ドラッグ vs クリック開始・終了)
- clipRange 実装の詳細(既存 jcut.json との互換性)
